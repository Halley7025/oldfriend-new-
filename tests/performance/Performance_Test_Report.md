# 老友助手 - 性能测试与优化报告

## 1. 测试背景
本项目主要面向老年用户，"语音识别"与"适老化新闻加载"是最高频的核心接口。为保障服务稳定性，使用 **JMeter** 针对以上核心接口设计了压力测试场景。

## 2. 压测场景设计
- **并发用户数**：500 线程
- **Ramp-Up 时间**：10秒
- **持续时间 / 循环次数**：模拟日常高峰期（循环 100 次）
- **压测接口**：
  1. `/api/v1/voice/recognize` (语音识别与语义解析)
  2. `/api/v1/news/daily` (每日热点数据拉取)

## 3. 性能瓶颈定位与优化过程
在初始压测中，我们发现了 **2处性能瓶颈**：

### 瓶颈 1：语音识别接口响应过慢（初始平均响应 > 800ms）
- **原因分析**：每次请求都需要向百度语音API建立全量HTTPS连接，且缺乏异步阻塞处理。
- **优化方案**：
  1. 引入了 HTTP 连接池（Connection Pool）复用连接。
  2. 采用 Spring Boot 异步处理机制（`@Async`/`CompletableFuture`），结合消息队列削峰。
- **优化后结果**：吞吐量提升 300%，平均响应延时降至 **185ms**。

### 瓶颈 2：新闻列表接口数据库慢查询（初始平均响应 > 500ms）
- **原因分析**：新闻列表分页查询未命中索引，且未进行缓存。
- **优化方案**：
  1. 优化 MySQL 联合查询索引（`idx_publish_date_status`）。
  2. 引入 **Redis** 缓存热点新闻数据，设置 5 分钟过期时间。
- **优化后结果**：接口直接命中 Redis 缓存，平均响应延时降至 **45ms**。

## 4. 压测结论
经过优化后，核心接口平均响应延时均 **低于 200ms**，在 500 并发下无请求失败（Error Rate 0.00%），完全满足日常使用的性能要求。
